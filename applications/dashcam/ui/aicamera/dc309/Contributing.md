
## lvgl 开发注意事项

- **尽量避免使用线程，优先使用 LVGL 的 timer**
  - LVGL 不是线程安全框架。
  - 多线程操作 LVGL 对象容易造成崩溃或 UI 异常。
  - 若必须跨线程更新 UI，应使用 **lv_async_call()** 或消息队列回到 LVGL 线程执行。

- **所有 LVGL 调用必须在同一个线程进行（一般是 GUI 主线程）**
  - 包含控件创建、删除、事件、样式操作。

- **避免频繁创建与销毁对象**
  - 可能导致内存碎片。
  - 尽量复用控件（如使用隐藏/显示代替重新创建）。

- **图片、字体等资源尽量静态加载**
  - 避免反复加载文件导致卡顿。
  - 大图片建议使用 **LV_IMG_CF_RAW / RAW_ALPHA**，提高加载速度。

- **定时器不要阻塞（timer 回调必须短）**
  - 避免在定时器里执行 IO、延迟、死循环。
  - 重活放到后台线程，再用 lv_async_call 回来更新 UI。

- **避免长时间阻塞 LVGL 的 task handler**
  - 主循环要持续执行 `lv_timer_handler()`。
  - 阻塞会导致卡顿、事件响应延迟。

- **动画数量不要过多**
  - 动画多会明显增加 CPU 占用。
  - 尽量合并动画或者减少动画刷新频率（use `lv_anim_set_delay()`）。

- **减少频繁的样式修改**
  - 多个样式建议预定义，不要在运行时频繁创建样式对象。
  - 样式操作比标签文本刷新更耗性能。

- **自定义控件尽量用已有基础组件组合实现**
  - 避免大量重绘影响性能。
  - 重新绘制（draw callback）应尽量简单。

- **大列表或图片使用虚拟化（virtual list/page）**
  - 避免加载大量对象导致内存爆掉。
  - 使用 LVGL 的 **tileview、list、table** 等控件的复用机制。

- **注意内存池大小（LV_MEM_SIZE）**
  - LVGL 内部分配是通过自定内存池，过小会导致 LVGL 报错或崩溃。
  - 复杂 UI 建议增加内存池。

- **事件处理**
  - 事件回调函数尽量只做状态更新，不做耗时操作。
  - 大量事件操作可使用消息队列或异步调用回到主线程处理。

- **日志与调试**
  - 在开发阶段开启调试日志，记录 UI 状态和事件触发情况。
  - UI 崩溃或卡顿时，优先检查：
    - 多线程操作 LVGL
    - Timer/动画阻塞
    - 内存池不足
    - 对象频繁创建/删除

- **命名与注释规范**
  - 控件变量示例：`lv_label_title`、`lv_btn_ok`
  - 样式变量示例：`style_screen_bg`、`style_btn_pressed`
  - 事件函数示例：`event_handler_btn_ok()`
  - 中文注释必须描述控件用途、事件逻辑和资源来源。

## 代码风格控制

使用**特定版本** `clang-format-12` 进行代码风格控制，配置文件位于项目根目录 `.clang-format`，完全使用 `WebKit` 风格，未做任何修改。

将下面的内容添加到 `.git/hooks/pre-commit` 文件中，可以在每次提交代码时自动格式化代码：

```bash
#!/bin/bash

# 获取当前暂存区中被修改的文件列表（只包括 .c/.cpp/.h/.hpp 文件）
files=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(c|cpp|h|hpp)$')

# 如果没有匹配文件，跳过
[ -z "$files" ] && exit 0

# 检查 clang-format-diff 是否存在
if ! command -v clang-format-diff-12 >/dev/null 2>&1; then
    echo "clang-format-diff not found. Please install it first."
    exit 1
fi

echo "Running clang-format-diff on staged C/C++ files..."

# 对每个文件应用 clang-format-diff
for file in $files; do
    # 忽略已删除的文件
    [ ! -f "$file" ] && continue

    # 获取当前暂存区与工作区的 diff
    git diff --cached "$file" | clang-format-diff-12 -p1 -i

    # 重新添加到暂存区（否则修改后提交的是旧版本）
    git add "$file"
done

echo "clang-format-diff applied successfully."

exit 0
```

## 代码规范

**全局变量**

1. 要明确区分全局变量是否是对外的，不是对外的全局变量必须用 `static` 修饰，防止命名冲突。
2. 对外的全局变量，要在头文件中使用 `extern` 声明，并在源文件中定义。
3. 全局变量命名要有前缀，避免命名冲突。例如：`g_` 表示全局变量，`s_` 表示静态变量
