## LVGL UI 应用开发

### 经验总结

- UI 一定要提前设计好，边写代码边设计UI，很浪费时间！！一定要明确需求！！
- 同理，写代码一定要写文档，写清楚设计思路，谁写代码谁写文档！！
- 在使用guiguider设计UI时，不要将所有控件都放一个全局变量中！！！！按页面设计结构体，分开管理。结构体太大不说，也不利于维护。
- **尽量**不要使用 magic number 来定义控件的大小！！！尤其是屏幕大小，应该使用宏定义来定义屏幕大小，方便移植到其他屏幕。


### 编译

- 使用 CMakelist.txt 文件编译，Makefile 仅提供快捷命令。
- 支持多种编译选项，支持交叉编译 和 目标平台设定。
- 使用了两个配置文件，SDL 使用 `config_sdl.h` 方便在PC中调试，FB 使用 `config_fb.h`。


### 日志

- `src/include/mlog.h` 文件中定义了日志记录的宏。
- 使用 `mlog` 进行日志记录，支持多种日志级别。
- 日志输出到控制台，直接修改 src/include/mlog.h 文件中的 `ML_LOG_LEVEL` 宏定义来设置日志级别。
- 放心使用 `MLOG_DBG` 增加调试日志输出，不要担心日志过得会臃肿，使用宏日志。
- 采用和 uboot 类似的方式，当全局日志级别定义为 INFO 或者更高时，在希望输出DEBUG 日志的文件最前面添加#define DEBUG，将输出该文件中的所有日志。


### 双缓冲避免屏幕撕裂

- 使用双缓冲技术来避免屏幕撕裂。


### 配置

- `src/include/config.h` 文件中定义了全局配置。
- 可以通过修改该文件来调整应用的全局配置。


## 注意：

- `ui_load_scr_animation` 函数的 `time`, `delay` 参数，前者是动画时间，可以让屏幕切换不那么生硬，在home1和home2切换时，time 设置为200 能体验到动画效果。`delay` 参数是延迟，最好不要设置，会给人一种响应速度慢的感觉。
- 为对象设置回调函数时，不要使用 EVENT_ALL，所有事件都走一下回调，这样会降低性能。也不方便查看DBG日志。
- 按页面设置回调函数，尽量将回调函数放在页面的 .c 文件中，不要全部放一起，页面过多之后会很臃肿，也不方便维护。

## UI 全局变量设计

- 使用一个全局变量 `lv_ui_t g_ui` 来管理所有 UI 控件。

```c
typedef struct
{
    lv_obj_t *scr; /* 页面容器：带4个按钮 + 底部页面标识 */
    bool del;
    lv_obj_t *imgbtn_album;
    lv_obj_t *imgbtn_setting;
    lv_obj_t *imgbtn_chinese;
} PageHome1_t; /* 注意命名格式 */

typedef struct
{
    /* 主界面1 */
    PageHome1_t page_home1;

    /* 主界面2 */
    PageHome2_t page_home2;
} lv_ui_t;
```

页面结构体：
- 按页面设计结构体，分开管理。（避免结构体太大，不利于维护）。
- ~~每个页面使用指针管理，页面创建、销毁时注意检查指针的状态。~~ 存在问题：
  - 判断时机问题：在 `ui_load_scr_animation` 中状态栏显示/隐藏判断在页面创建之前执行
  - 容易出错，不利于维护，需要检查空指针
- 页面还是用成员变量，改为指针动态分配、创建会带来很多问题，容易出错，不利于维护，虽然这样总的结构体变大了，但先这样吧。
  - 注意：page_home1 这种子结构体，成员仍然用指针，否则每个页面，每个obj都常驻内存的话，占用内存太大了。也不合理。
  - **对象销毁同步问题**：LVGL会自动销毁屏幕对象，但不会重置我们的指针。解决方案：使用 `lv_obj_is_valid()` 检查对象有效性，避免删除已销毁的对象。
- 成员命名要有意义，格式为小写+下划线。带有对象类型信息，比如 `lv_imagebutton_create` 创建的对象，使用 imgbtn_xxx 命名。

## 音量控件设计思路

- 实现在任意页面按下音量按键时，都能显示音量控件。并2s后自动隐藏。
- 放在 top layer。因为音量控件需要显示在所有页面的最上层。


## 顶部状态栏设计思路

- 顶部状态栏包含时间、wifi信号、4G信号、电量等信息。
- 状态栏实时更新，显示当前时间、wifi信号强度、4G信号强度和电量。
- 放在 top layer。因为状态栏需要创建定时更新时间、wifi信号、4G信号、电量等，两个页面都需要用到，放 top layer 统一使用同一个对象，方便处理时间更新等回调。


## Home1 页面设计思路

内容：
- 顶部状态栏（top layer，可见）
- 拍万物
- 拍照（传统拍照、录像）
- AI对话
- AI特效
- 底部页面标识（当前页面，这个很简单的控件，也只有两个home页面用到，就不放到 top layer 层了）
支持动作：
- 顶部状态栏实时更新 时间、wifi信号、4g信号、电量
- 拍照、录像、AI对话、AI特效 **按键点击**后，分别跳转到对应的页面。
- 从右向左滑动，可切换到 Home2 页面。

切换到其他页面时，是否销毁？
- Home1 页面切换到其他页面时，不销毁。原因：两个Home页面作为常用页面，应该保持常驻内存，避免频繁创建和销毁。
